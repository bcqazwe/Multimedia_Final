# Proposal: boss-three-phase-attack-design

## 概要
本變更將現有單一 Boss 戰擴充為三階段戰鬥流程，每個階段配置 3 種攻擊方式，總計 9 種攻擊。設計重點不是單純增加招式數量，而是讓 Boss 在後續階段變得更有侵略性、更少空檔、更強的區域壓迫感，形成清楚的節奏曲線。

## 為何要做這個變更
- 目前原型中的 Boss 主要只有水平移動與基礎血量顯示，戰鬥節奏偏單薄。
- 規格已要求 Boss 具備多階段血量或多段攻擊模式，因此需要補上可執行的攻擊框架。
- 三階段結構能自然對應現有的 3000 HP 設計，讓 1000 HP 一段成為清楚的節奏切點。

## 範圍
- 包含：Boss 三階段狀態機、9 種攻擊模式、階段切換、攻擊前搖與後搖、視覺提示、侵略性遞增規則。
- 包含：與現有 `main.py`、`Boss.py`、`Health.py` 的整合設計。
- 不包含：音訊、第二個 Boss、多波小怪、完整粒子特效系統。

## 預期成果
- Boss 擁有 Phase 1 / Phase 2 / Phase 3 的清楚戰鬥節奏。
- 每個階段至少有 3 種可辨識攻擊，且越後面階段越快、越密、越難閃。
- 任何階段切換都可由血量直接推導，便於平衡與測試。
