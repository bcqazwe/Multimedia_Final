# Tasks: boss-three-phase-attack-design

## 目標
把三階段 Boss 攻擊設計拆成可實作任務，先做骨架，再做攻擊家族，最後做平衡與驗證。

### 1. Phase 系統骨架
- 建立 Boss phase 判定，依 3000 / 2000 / 1000 HP 切換。
- 定義 Boss 狀態機：進場、前搖、攻擊、後搖、轉階段、擊敗。
- 確認 HP 歸零後會切到 WIN，而不是繼續執行 Boss 行為。

### 2. 攻擊家族 A：彈幕型
- 實作直線掃射。
- 實作交叉扇形。
- 實作高速鎖定彈。
- 依階段調整彈速、數量與前搖。

### 3. 攻擊家族 B：位移型
- 實作慢速橫移。
- 實作短衝接回位。
- 實作連段突進。
- 讓位移與彈幕能串接成連招。

### 4. 攻擊家族 C：場地壓制型
- 實作單點落彈。
- 實作雙點交錯落彈。
- 實作區域封鎖大招。
- 加入地面警示或危險區提示。

### 5. UI 與提示
- 保留現有 Boss 血條分段顯示。
- 在必要時顯示 phase 轉換或危險提示。
- 確保 WIN 畫面與返回主選單流程不受影響。

### 6. 平衡與測試
- 為三個階段分別測試攻擊間隔、密度、前搖與可閃避性。
- 在每個階段驗證玩家是否仍有清楚的安全窗口。
- 記錄 FPS 與主觀難度回饋，調整攻擊參數。

## 驗收條件
- Boss 具有三個可辨識階段。
- 每個階段都有至少 3 種不同攻擊。
- Phase 越後面，攻擊節奏越快、壓力越高。
- Boss 歸零後仍正常進入 WIN 畫面。
- 返回主選單後遊戲可以正常重置並重新開局。
