(已完成，不須實作)Multimedia_Final 優化方法整理

一、背景與渲染優化
1. 背景顯示快取
- 將背景圖的 resize 結果快取成顯示尺寸畫面。
- 避免每一幀都重新 resize，降低 CPU 負擔。

2. Sprite 顏色與遮罩快取
- 將 boss、玩家子彈、boss 子彈、警示圖示拆成 RGB 與 Alpha mask。
- 減少每幀重複切片與重複處理透明度。

3. OpenCV C 層混合
- 以 cv2.multiply 與 cv2.add 取代 Python 層的逐像素混合。
- 用 C 層運算降低 alpha 合成成本。

二、物件與彈幕優化
4. 物件池（Object Pool）
- 玩家子彈與 boss 子彈改用 pool 重用物件。
- 減少大量建立與銷毀 dict 造成的記憶體配置成本。

5. 提早回收離場物件
- 子彈離開畫面後較早回收。
- 降低 update / draw 時的無效物件數量。

6. 彈幕生成與難度調整
- boss 子彈數量與速度提高。
- 交叉扇形角度往下偏，讓彈幕更貼近玩家活動區。
- 第 2、3 階段進一步增加速度與發射頻率。

三、碰撞與遊戲邏輯優化
7. 碰撞檢查節流
- 將 combat check 改為每 32 ms 執行一次。
- 降低每幀大量碰撞判斷的 CPU 成本。

8. 攻擊判定區域簡化
- 玩家核心區、boss rect、rocket zone 以簡化矩形判斷為主。
- 減少複雜幾何運算。

四、Boss 與關卡調整
9. Boss 血量與分段
- Boss 總血量調整為 9999。
- 三階段以總血量平均三等分切換。

10. Boss 圖示與視覺調整
- boss_bullet_dot 放大，提升辨識度。
- boss 子彈與攻擊圖示整體更醒目。

五、效能結果
- 背景 resize 成本下降。
- alpha 合成改為 C 層運算。
- 子彈建立與回收成本下降。
- 每幀碰撞檢查次數下降。
- 整體畫面與彈幕更新更順暢。

六、後續可延伸但尚未正式納入
- 類別批次渲染：同類子彈先畫到臨時畫布，再一次疊回主畫面。
- 更完整的 benchmark：記錄每幀耗時，量化每項優化效果。
- 模式參數化：把 boss 子彈速度、數量、間隔集中成可切換設定表。
